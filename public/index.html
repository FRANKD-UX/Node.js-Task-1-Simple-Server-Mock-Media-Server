Media API Documentation

This API provides endpoints to manage movies, TV series, and songs. Use the endpoints below to interact with the different media collections.

Endpoints Overview

  • /movies - Manage movie collection
  • /series - Manage TV series collection
  • /songs - Manage song collection

Movies API

GET /movies

Returns a list of all movies in the collection.

Response

[
  {
    "id": 1,
    "title": "Inception",
    "director": "Christopher Nolan",
    "year": 2010,
    "genre": "Sci-Fi",
    "rating": 8.8
  },
  ...
]
POST /movies

Adds a new movie to the collection.

Request Body

{
  "title": "New Movie Title",
  "director": "Director Name",
  "year": 2023,
  "genre": "Genre",
  "rating": 8.5
}

Response

Returns the updated movie collection with the new movie included, with an auto-generated ID.

PUT /movies

Updates an existing movie in the collection.

Request Body

{
  "id": 1,
  "title": "Updated Movie Title",
  "director": "Director Name",
  "year": 2023,
  "genre": "Genre",
  "rating": 8.5
}

Response

Returns the updated movie collection.

DELETE /movies

Deletes a movie from the collection by ID.

Request Body

{
  "id": 1
}

Response

Returns the updated movie collection with the specified movie removed.

Error Handling

The API returns appropriate HTTP status codes along with error messages for various scenarios:

  • 400 Bad Request - For invalid input data
  • 404 Not Found - When an item or resource is not found
  • 405 Method Not Allowed - When using an unsupported HTTP method
  • 500 Internal Server Error - For server-side errors
